Only 8 queries, so the attention matrix is a thin 8×217 band, not a 217×217 square

Self-attention over the tokens would score every token against every token, a full 217×217 square. Here 8 learned queries score against the 217 keys, so the matrix collapses to 8 rows. The block reads all tokens but keeps only 8 outputs.

How to read it. The 217 context tokens (196 vision, 8 proprioception, 13 end-effector, width d = 256) are projected once into keys and values. The scores are a matrix multiply, Q (8×256) · Kᵀ (256×217) = A (8×217). Because there are only 8 queries, A is a thin band of 8 rows, not the 217×217 square that self-attention over the same tokens would build (the dashed outline). Each row still spans all 217 tokens (the sweep). Then A (8×217) · V (217×256) = O (8×256) returns one vector per query, so the token count drops out. Cost is O(N M) rather than O(M²). The block repeats twice, then the 8 rows are mean-pooled into z ∈ ℝ²⁵⁶.
